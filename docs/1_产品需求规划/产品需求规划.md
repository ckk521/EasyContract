# 产品需求规划

> AI智能合同审核平台 - 智能体 + B端配置驱动的合同审核系统

---

## 1. 产品定位

**一句话定位**：为律所律师提供智能合同生成与审核服务，通过AI + 人工协同，将合同处理时间从小时级缩短到分钟级。

**核心价值**：
- **效率提升**：智能体自动收集需求、生成合同、审核风险，减少80%重复性工作
- **质量保障**：AI辅助风险识别，降低法律风险，提升合同质量
- **服务升级**：快速响应客户需求，提升客户满意度与粘性

**成功指标**（MVP目标）：
- DAU：50人（单个律所试点）
- 合同生成时间：≤15分钟
- 合同审核时间：≤5分钟
- 风险识别准确率：≥85%

## 4. 核心业务场景

### 场景1：合同生成

**用户旅程**：
```
用户描述需求 → 智能体提问 → 用户回答 → 生成合同 → 用户确认 → 导出文档
```

**详细流程**：
1. 用户通过聊天界面描述合同需求（如"我需要一份房屋租赁合同"）
2. 智能体识别意图，调用合同生成流程
3. 智能体根据合同类型，逐项提问收集必要信息
4. 用户回答问题
5. 智能体调用合同模板，填充信息，生成合同初稿
6. 用户预览合同，可进行修改或直接导出

**成功标准**：
- 从需求到初稿时间 ≤15分钟
- 信息收集准确率 ≥90%

### 场景2：法律咨询

**用户旅程**：
```
用户提问 → 智能体判断 → RAG检索 → 返回答案
```

**详细流程**：
1. 用户提问（如"违约责任怎么约定"）
2. 智能体判断是否属于法律范畴
3. 非法律问题委婉拒绝，引导回法律范畴
4. 法律问题通过RAG检索知识库
5. 返回答案（条款解释、案例参考）

**成功标准**：
- 响应时间 ≤3秒
- 答案准确率 ≥90%

### 场景3：合同审核（V1.1）

**用户旅程**：
```
C端提交 → 律师分配 → AI初审 → 主管审批 → 完成
```

**详细流程**：
1. C端用户提交合同审核
2. 系统生成律师推荐列表（按专长匹配度排序）
3. admin手动或系统自动分配律师
4. 律师执行AI审核
5. 律师提交主管审批
6. 主管审批通过/驳回

## 5. 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| C端用户（咨询人员） | 需要法务咨询的客户/咨询人员 | 快速获取法务咨询、智能问答、合同生成 |
| B端admin | 系统管理员 | 管理用户、配置智能体、上传规则 |
| B端主管 | 审批下属律师提交的合同 | 审批/驳回合同、管理下属 |
| B端律师 | 审核分配给自己的合同 | AI审核、提交审批 |

## 6. 合同状态流转

### C端合同状态
| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| draft | 草稿 | 编辑、删除、提交审核 |
| pending_review | 待审核 | AI审核、分配审核人 |
| ai_reviewed | AI已初审 | 人工审核、查看AI报告 |
| in_review | 审核中 | 审核、驳回（仅员工） |
| rejected | 已驳回 | admin审核、查看驳回原因 |
| completed | 已完成 | 查看、导出 |

### B端审核流程
```
draft → pending_review → in_review → ai_reviewed → pending_supervisor → completed
                              ↓              ↓
                          rejected      (驳回返回in_review)
```

## 8. 核心设计原则

> **配置驱动架构**：C端智能体的所有行为都依赖B端配置，B端运营人员可自由调整Agent策略，无需修改代码。

```
B端配置中心 → 配置存储 → Agent运行时
     ↑                          ↓
     └──── 配置生效反馈 ←───────┘
```

**配置实时生效机制**：
- 所有配置实时生效，无需重启服务
- 配置变更时自动更新RAG索引
- 配置历史记录保留，支持回滚
