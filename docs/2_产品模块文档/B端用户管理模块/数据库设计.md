# B端用户管理模块 - 数据库设计

## 1. staff - B端员工表

```sql
CREATE TABLE staff (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(100) UNIQUE NOT NULL,   -- 用户名
    password VARCHAR(255) NOT NULL,           -- bcrypt加密
    nickname VARCHAR(50),                     -- 昵称
    role VARCHAR(20) NOT NULL DEFAULT 'lawyer',  -- admin/supervisor/lawyer
    status VARCHAR(20) NOT NULL DEFAULT 'active',  -- active/inactive
    first_login BOOLEAN DEFAULT 1,           -- 是否首次登录
    supervisor_id INTEGER,                    -- 所属主管ID（律师专用）
    work_status VARCHAR(20) DEFAULT 'available',  -- available/pto

    -- 职业信息（律师专用）
    license_number VARCHAR(50),              -- 执业证号
    years_of_practice INTEGER,               -- 执业年限
    employee_id VARCHAR(50),                 -- 工号
    title VARCHAR(50),                       -- 职称
    law_firm VARCHAR(100),                   -- 所属律所

    -- 专业能力（律师专用）
    education VARCHAR(20),                   -- 学历
    law_school VARCHAR(100),                 -- 毕业院校
    certifications JSON,                       -- 专业资质
    languages JSON,                           -- 语言能力

    -- 专长配置
    specialties JSON,                        -- 合同类型专长配置

    last_login_at DATETIME,
    last_login_ip VARCHAR(50),
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (supervisor_id) REFERENCES staff(id)
);

-- 索引
CREATE UNIQUE INDEX idx_staff_username ON staff(username);
CREATE INDEX idx_staff_role ON staff(role);
CREATE INDEX idx_staff_status ON staff(status);
CREATE INDEX idx_staff_supervisor ON staff(supervisor_id);
```

### specialties 字段格式

```json
{
  "contract_types": {
    "租赁合同": "expert",
    "买卖合同": "skilled"
  }
}
```

### certifications 字段格式

```json
["专利代理师", "注册会计师", "税务师"]
```

### languages 字段格式

```json
[
  {"language": "英语", "level": "流利"},
  {"language": "日语", "level": "基础"}
]
```

## 2. messages - 站内信消息表

```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    recipient_id INTEGER NOT NULL,
    recipient_type VARCHAR(20) NOT NULL DEFAULT 'staff',  -- 接收人类型
    message_type VARCHAR(50) NOT NULL,                   -- 消息类型
    title VARCHAR(200) NOT NULL,                         -- 消息标题
    content TEXT,                                        -- 消息内容
    related_id INTEGER,                                  -- 关联业务ID（如合同ID）
    related_type VARCHAR(50),                            -- 关联业务类型
    is_read BOOLEAN DEFAULT FALSE,                        -- 是否已读
    read_at DATETIME,                                    -- 阅读时间
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_recipient ON messages(recipient_id, recipient_type);
CREATE INDEX idx_is_read ON messages(is_read);
CREATE INDEX idx_created_at ON messages(created_at);
CREATE INDEX idx_message_type ON messages(message_type);
```

### 消息类型定义

| 消息类型 | 触发场景 | 接收者 | 标题示例 |
|---------|---------|--------|---------|
| contract_assigned | 合同分配给律师 | 律师 | "您有新的合同任务待处理" |
| lawyer_submitted | 律师提交审批 | 主管 | "有新的合同待审批" |
| supervisor_approved | 主管审批通过 | 律师 | "您的合同审批已通过" |
| supervisor_rejected | 主管审批驳回 | 律师 | "您的合同被驳回" |
| manual_assign_needed | 需要手动分配 | admin | "有合同需要手动分配律师" |
| new_contract_submitted | C端用户提交审核 | admin | "收到新合同审核申请" |
| system_notice | 系统公告 | 全部B端用户 | "系统维护通知" |

## 3. 合同表扩展 (contracts)

```sql
-- 分配相关
ALTER TABLE contracts ADD COLUMN assigned_lawyer_id INTEGER;     -- 分配的律师ID
ALTER TABLE contracts ADD COLUMN assigned_by INTEGER;           -- 分配人ID（手动分配时为admin，自动分配时为null）
ALTER TABLE contracts ADD COLUMN assigned_at DATETIME;           -- 分配时间
ALTER TABLE contracts ADD COLUMN assign_type VARCHAR(20);       -- 分配方式：auto/manual

-- 主管审批相关
ALTER TABLE contracts ADD COLUMN supervisor_reviewed_by INTEGER;  -- 主管审批人ID
ALTER TABLE contracts ADD COLUMN supervisor_reviewed_at DATETIME; -- 主管审批时间
ALTER TABLE contracts ADD COLUMN supervisor_rejected_at DATETIME; -- 主管驳回时间
ALTER TABLE contracts ADD COLUMN supervisor_reject_reason VARCHAR(500); -- 主管驳回原因
```

## 4. 审核日志表 (contract_review_logs)

```sql
CREATE TABLE contract_review_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    contract_id INTEGER NOT NULL,
    action VARCHAR(50) NOT NULL,       -- ai_review/assign/lawyer_submit/supervisor_approve/supervisor_reject/admin_complete
    operator_id INTEGER NOT NULL,
    operator_role VARCHAR(20),         -- admin/supervisor/lawyer
    detail JSON,                       -- 操作详情
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (contract_id) REFERENCES contracts(id)
);
```

### action 字段值

| action | 说明 | from_status | to_status | 操作人 |
|--------|------|-------------|-----------|--------|
| submit_review | 提交审核 | draft | pending_review | C端用户 |
| auto_assign | 自动分配律师 | pending_review | in_review | 系统 |
| manual_assign | 手动分配律师 | pending_review | in_review | admin |
| ai_review_execute | 律师执行AI审核 | in_review | ai_reviewed | 律师 |
| lawyer_submit | 律师提交审批 | ai_reviewed | pending_supervisor | 律师 |
| supervisor_approve | 主管通过 | pending_supervisor | completed | 主管 |
| supervisor_reject | 主管驳回 | pending_supervisor | in_review | 主管 |
| admin_complete | admin直接完成 | * | completed | admin |
