## 1. Agent核心架构

### 1.1 决策循环

```
用户输入 → InputProcessor → IntentClassifier → Router → ToolExecutor → ResponseGenerator → 用户输出
```

**组件职责：**

| 组件 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **InputProcessor** | 文本预处理 | 原始用户输入 | 清洗后的文本 + 实体列表 |
| **IntentClassifier** | 意图识别 | 清洗后文本 | IntentResult(类型, 置信度, 参数) |
| **Router** | 路由分发 | IntentResult | 处理策略 |
| **ToolExecutor** | 工具执行 | 工具名+参数 | 工具返回结果 |
| **ResponseGenerator** | 响应生成 | 处理结果 | 自然语言响应 |

### 1.2 状态机

```
IDLE → INTENT_RECOGNIZING → ROUTING → TOOL_EXECUTING → RESPONDING
                                         ↓
                              ┌──────────┴──────────┐
                              ↓                      ↓
                        FORM_FILLING            COMPLETED
                              ↓
                        DRAFT_SAVED
```

## 3. 置信度策略

### 3.1 什么是置信度？

置信度是意图识别系统对判断结果的**可信程度**（0~1之间的数值）：

| 置信度 | 含义 | 举例 |
|--------|------|------|
| **0.9~1.0** | 非常有把握 | "我要租房" → CONTRACT_DRAFT + RENTAL |
| **0.7~0.9** | 比较有把握 | "帮我弄个合同" → CONTRACT_DRAFT |
| **0.5~0.7** | 不太确定 | "有个事想咨询" → GENERAL_CHAT? |
| **< 0.5** | 很不确定 | "你好" → 无法判断 |

### 3.2 置信度阈值策略

| 置信度范围 | 系统行为 |
|-----------|----------|
| **>= 0.85** | 高置信度，直接执行对应意图 |
| **0.6 ~ 0.85** | 中置信度，执行意图但添加确认步骤 |
| **0.4 ~ 0.6** | 低置信度，要求用户澄清 |
| **< 0.40** | 极低置信度，切换到 GENERAL_CHAT |

### 3.3 置信度计算方式

**混合策略（Hybrid）：**

1. **规则匹配（快速路径）**
   - 命中关键词 = 高置信度 (0.9)
   - 例："租房" + "合同" → CONTRACT_DRAFT

2. **语义相似度（中等成本）**
   - 计算用户输入与预设意图模板的相似度
   - 相似度 > 0.8 = 高置信度

3. **LLM分类（精准但慢）**
   - 用小模型或主模型 + 分类prompt
   - 直接输出意图 + 置信度

```python
# 示例：置信度计算伪代码
def calculate_confidence(user_input: str) -> tuple[Intent, float]:
    # 1. 规则匹配
    for rule in intent_rules:
        if rule.matches(user_input):
            return rule.intent, 0.95  # 规则匹配 = 高置信度

    # 2. 语义相似度
    similarity = calculate_similarity(user_input, intent_templates)
    if similarity > 0.8:
        return similarity.best_intent, similarity.score

    # 3. LLM分类（兜底）
    return llm_classify(user_input)
```

### 3.4 用户体验示例

**高置信度 (>= 0.85)：**
```
用户: "我要租房"
系统: 您想起草租赁合同，我来帮您找模板...
```

**中置信度 (0.6~0.85)：**
```
用户: "帮我弄个租房的东西"
系统: 您是想起草租房相关的合同吗？我这里有房屋租赁合同模板...
[用户确认] → 执行
```

**低置信度 (< 0.6)：**
```
用户: "你好啊"
系统: 您好！我是法律助手，请问有什么可以帮助您的？
[系统要求用户澄清意图]
```

#### 4.3.3 完整 ReAct Agent 实现

```python
# agent/react_agent.py
from dataclasses import dataclass, field
from typing import TypedDict, Annotated, Sequence
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langchain_openai import ChatOpenAI
from langchain.agents import create_openai_functions_agent, AgentExecutor

class AgentState(TypedDict):
    """Agent 状态"""
    messages: Annotated[Sequence[BaseMessage], add_messages]
    intent: str | None
    sub_intent: str | None
    confidence: float
    entities: dict
    plan: list[dict]           # 工具执行计划
    execution_results: list    # 执行结果
    current_step: int

class ReActAgent:
    """
    ReAct Agent - 可扩展版

    扩展方式：
    1. 新增工具 → 在 tools/ 目录用 @registry.register 注册
    2. 新增意图 → 在 intent_templates.json 添加模板
    3. 新增分类 → 在 ToolCategory 添加枚举值
    """

    def __init__(
        self,
        registry: 'ToolRegistry',
        executor: 'ToolExecutor',
        intent_classifier: 'IntentClassifier',
    ):
        self.registry = registry
        self.executor = executor
        self.intent_classifier = intent_classifier
        self.llm = ChatOpenAI(model="minimax-m2.7", temperature=0.7, streaming=True)

        # 构建 Agent
        self.agent = create_openai_functions_agent(
            llm=self.llm,
            tools=self.registry.get_langchain_tools(),  # 转换为 LangChain Tools
            prompt=self._build_prompt(),
        )
        self.agent_executor = AgentExecutor(
            agent=self.agent,
            tools=self.registry.get_langchain_tools(),
            verbose=True,
            max_iterations=10,
        )

    def _build_prompt(self) -> str:
        return """你是一个专业的法律助手，名为"法小才"。

你的职责：
1. 帮助用户起草各类合同（租赁、劳动、买卖等）
2. 回答用户的法律问题
3. 引导用户填写合同信息并保存草稿

工作流程：
1. 理解用户需求（意图识别）
2. 调用相应工具获取信息
3. 用友好、专业的语言回复用户
4. 对于非法律问题，委婉拒绝并引导回归法律范畴

记住：
- 每次回复要简洁、专业、友好
- 支持流式输出（SSE）给用户更好的体验
- 调用工具后，基于工具返回结果回复用户
"""

    async def process(self, user_input: str, user_context: dict) -> dict:
        """处理用户输入"""
        # 意图识别
        intent_result = await self.intent_classifier.classify(user_input)

        # 构建初始状态
        state = AgentState(
            messages=[HumanMessage(content=user_input)],
            intent=intent_result.intent,
            sub_intent=intent_result.sub_intent,
            confidence=intent_result.confidence,
            entities=intent_result.entities,
            plan=[],
            execution_results=[],
            current_step=0,
        )

        # 执行 ReAct 循环
        if intent_result.confidence < 0.4:
            return await self._generate_rejection(state)

        return await self._react_loop(state, user_context)

    async def _react_loop(self, state: AgentState, user_context: dict) -> dict:
        """ReAct 主循环"""
        max_iterations = 10

        while state["current_step"] < max_iterations:
            current_message = state["messages"][-1].content

            # 调用 Agent（内部处理 Thought/Action/Observation）
            response = await self.agent_executor.ainvoke({
                "input": current_message,
                "chat_history": state["messages"][:-1],
                "user_context": user_context,
            })

            # 检查是否需要继续循环
            if not response.get("tool_calls"):
                break

            state["messages"].append(AIMessage(content=response["output"]))
            state["current_step"] += 1

        return {
            "response": response.get("output", ""),
            "intent": state["intent"],
            "success": True,
        }

    async def _generate_rejection(self, state: AgentState) -> dict:
        """生成拒绝回复"""
        # 从配置获取拒绝话术
        rejection_prompt = config.get("rejection.prompt",
            "抱歉，我是法律助手，专注于为您提供法律服务哦~\n请问有什么法律问题可以帮到您？"
        )
        return {
            "response": rejection_prompt,
            "intent": "REJECTED",
            "confidence": state["confidence"],
        }
```

```python
# tool_registry.py
from dataclasses import dataclass, field
from typing import Callable, Any
from enum import Enum

class ToolCategory(Enum):
    """工具分类"""
    CONTRACT = "contract"           # 合同相关
    LEGAL = "legal"                 # 法律咨询
    REVIEW = "review"               # 审核相关
    USER = "user"                   # 用户管理
    UTILITY = "utility"             # 通用工具

@dataclass
class ToolMetadata:
    """工具元数据"""
    name: str
    category: ToolCategory
    description: str
    version: str = "1.0.0"
    enabled: bool = True
    intents: list[str] = field(default_factory=list)
    parameters: dict = field(default_factory=dict)
    examples: list[str] = field(default_factory=list)
    timeout_ms: int = 30000
    cacheable: bool = False

class ToolRegistry:
    """
    工具注册中心 - 核心扩展机制

    使用方式：
    @registry.register(category=ToolCategory.CONTRACT, intents=["CONTRACT_DRAFT"])
    def template_search(params):
        ...
    """
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._tools = {}
            cls._instance._categories = {}
        return cls._instance

    def register(
        self,
        category: ToolCategory,
        intents: list[str] = [],
        version: str = "1.0.0",
        description: str = "",
        parameters: dict = None,
        examples: list[str] = None,
        timeout_ms: int = 30000,
        cacheable: bool = False,
    ):
        """装饰器注册工具"""
        def decorator(func: Callable) -> Callable:
            metadata = ToolMetadata(
                name=func.__name__,
                category=category,
                description=description,
                version=version,
                intents=intents,
                parameters=parameters or {},
                examples=examples or [],
                timeout_ms=timeout_ms,
                cacheable=cacheable,
            )

            # 注册到分类索引
            if category.value not in self._categories:
                self._categories[category.value] = []
            self._categories[category.value].append(metadata.name)

            # 注册工具
            self._tools[func.__name__] = {
                "func": func,
                "metadata": metadata,
            }
            return func
        return decorator

    def get_tool(self, name: str) -> tuple[Callable, ToolMetadata] | None:
        """获取工具及其元数据"""
        tool = self._tools.get(name)
        if tool:
            return tool["func"], tool["metadata"]
        return None

    def get_tools_by_intent(self, intent: str) -> list[tuple[Callable, ToolMetadata]]:
        """根据意图获取可用工具"""
        results = []
        for name, tool in self._tools.items():
            if intent in tool["metadata"].intents or not tool["metadata"].intents:
                results.append((tool["func"], tool["metadata"]))
        return results

    def get_langchain_tools(self) -> list:
        """获取 LangChain Tools 格式"""
        from langchain_core.tools import tool
        result = []
        for name, tool_data in self._tools.items():
            metadata = tool_data["metadata"]
            if metadata.enabled:
                func = tool_data["func"]
                # 转换已有函数为 LangChain tool（实际使用中需要适当包装）
                result.append(func)
        return result

    def list_all_tools(self) -> list[ToolMetadata]:
        """列出所有已注册工具"""
        return [t["metadata"] for t in self._tools.values()]

    def enable(self, name: str, enabled: bool = True):
        """启用/禁用工具"""
        if name in self._tools:
            self._tools[name]["metadata"].enabled = enabled

# 全局注册中心
registry = ToolRegistry()
```

### 5.2 意图-工具自动路由

```python
# intent_router.py
from typing import list
from dataclasses import dataclass

@dataclass
class IntentResult:
    intent: str
    sub_intent: str | None
    confidence: float
    entities: dict
    reasoning: str
    suggested_tools: list[str]  # 自动从 registry 获取

class AutoIntentRouter:
    """
    自动意图路由 - 根据识别的意图自动选择工具

    优势：
    - 新增工具无需修改路由代码
    - 工具与意图解耦
    """

    def __init__(self, registry: ToolRegistry):
        self.registry = registry

    def route(self, intent_result: IntentResult) -> list[dict]:
        """根据意图返回应调用的工具列表"""
        # 从注册中心获取适用意图
        available_tools = self.registry.get_tools_by_intent(intent_result.intent)

        # 根据子意图进一步筛选
        if intent_result.sub_intent:
            available_tools = [
                (func, meta) for func, meta in available_tools
                if self._match_sub_intent(meta, intent_result.sub_intent)
            ]

        # 按版本排序
        available_tools.sort(key=lambda x: x[1].version, reverse=True)

        # 构建工具调用计划
        plan = []
        for func, metadata in available_tools:
            plan.append({
                "tool_name": metadata.name,
                "description": metadata.description,
                "parameters": metadata.parameters,
                "timeout_ms": metadata.timeout_ms,
                "cacheable": metadata.cacheable,
            })

        return plan

    def _match_sub_intent(self, metadata: ToolMetadata, sub_intent: str) -> bool:
        """匹配子意图"""
        params = metadata.parameters.get("properties", {})
        if "template_type" in params:
            enum_values = params["template_type"].get("enum", [])
            return sub_intent in enum_values or not enum_values
        return True
```

### 5.3 工具执行器

```python
# tool_executor.py
import asyncio
from typing import Any, Callable
from dataclasses import dataclass
from enum import Enum
import time
import json

class ExecutionStatus(Enum):
    SUCCESS = "success"
    FAILED = "failed"
    TIMEOUT = "timeout"
    DISABLED = "disabled"

@dataclass
class ExecutionResult:
    tool_name: str
    status: ExecutionStatus
    result: Any
    error: str | None
    duration_ms: int
    cached: bool = False

class ToolExecutor:
    """
    工具执行器 - 负责安全执行工具调用
    """

    def __init__(self, registry: ToolRegistry, cache=None):
        self.registry = registry
        self.cache = cache  # Redis 缓存
        self._execution_history = []

    async def execute(
        self,
        tool_name: str,
        parameters: dict,
        user_context: dict = None
    ) -> ExecutionResult:
        """执行单个工具"""
        start_time = time.time()

        # 1. 获取工具
        tool_func, metadata = self.registry.get_tool(tool_name)
        if not tool_func:
            return ExecutionResult(
                tool_name=tool_name,
                status=ExecutionStatus.FAILED,
                result=None,
                error=f"Tool {tool_name} not found",
                duration_ms=0,
            )

        # 2. 检查是否启用
        if not metadata.enabled:
            return ExecutionResult(
                tool_name=tool_name,
                status=ExecutionStatus.DISABLED,
                result=None,
                error="Tool is disabled",
                duration_ms=0,
            )

        # 3. 检查缓存
        if metadata.cacheable and self.cache:
            cache_key = f"tool:{tool_name}:{hash(str(parameters))}"
            cached_result = await self.cache.get(cache_key)
            if cached_result:
                return ExecutionResult(
                    tool_name=tool_name,
                    status=ExecutionStatus.SUCCESS,
                    result=json.loads(cached_result),
                    duration_ms=0,
                    cached=True,
                )

        # 4. 执行（带超时）
        try:
            if asyncio.iscoroutinefunction(tool_func):
                result = await asyncio.wait_for(
                    tool_func(parameters, user_context),
                    timeout=metadata.timeout_ms / 1000
                )
            else:
                result = tool_func(parameters, user_context)

            duration_ms = int((time.time() - start_time) * 1000)

            # 5. 写入缓存
            if metadata.cacheable and self.cache:
                await self.cache.setex(cache_key, 3600, json.dumps(result))

            # 6. 记录执行历史
            self._execution_history.append({
                "tool": tool_name,
                "status": "success",
                "duration_ms": duration_ms,
                "timestamp": time.time(),
            })

            return ExecutionResult(
                tool_name=tool_name,
                status=ExecutionStatus.SUCCESS,
                result=result,
                error=None,
                duration_ms=duration_ms,
            )

        except asyncio.TimeoutError:
            return ExecutionResult(
                tool_name=tool_name,
                status=ExecutionStatus.TIMEOUT,
                result=None,
                error=f"Tool execution timeout after {metadata.timeout_ms}ms",
                duration_ms=metadata.timeout_ms,
            )
        except Exception as e:
            return ExecutionResult(
                tool_name=tool_name,
                status=ExecutionStatus.FAILED,
                result=None,
                error=str(e),
                duration_ms=int((time.time() - start_time) * 1000),
            )
```

## 6. 记忆层设计 (Memory)

### 6.1 记忆分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                     AGENT MEMORY                            │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐      │
│  │   SESSION   │   │    USER     │   │   ENTITY    │      │
│  │   MEMORY    │   │   PROFILE   │   │   MEMORY    │      │
│  │  (当前对话)  │   │  (用户画像)  │   │  (实体信息)  │      │
│  └─────────────┘   └─────────────┘   └─────────────┘      │
│                           │                                │
│                           └────────────┬───────────────────┘
│                                        ▼                    │
│                             ┌─────────────────────────┐     │
│                             │     VECTOR STORE        │     │
│                             │   (语义记忆/历史检索)     │     │
│                             └─────────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 各层职责

| 记忆层 | 生命周期 | 存储位置 | 说明 |
|--------|----------|----------|------|
| **Session Memory** | 当前会话 | Redis + SQLite | 当前对话上下文，随会话结束清除 |
| **User Profile** | 长期 | SQLite | 用户偏好、常用合同类型 |
| **Entity Memory** | 中期 | SQLite | 从对话中提取的关键实体 |
| **Vector Store** | 长期 | SQLite/向量库 | 历史对话的语义索引 |

### 6.3 Session Memory（会话记忆）

当前对话的上下文状态：

| 数据字段 | 说明 | 示例 |
|----------|------|------|
| `current_intent` | 当前意图 | CONTRACT_DRAFT |
| `collected_fields` | 已收集的字段 | {lessor: "张三", lessee: "李四"} |
| `template_id` | 选定的模板ID | 5 |
| `pending_fields` | 待填写字段 | ["rent", "deposit"] |
| `conversation_turns` | 对话轮次 | 4 |

**读写流程：**
```
用户: "我要租朝阳区100平米的房子"
→ IntentClassifier 识别意图
→ Session Memory 更新: {current_intent: CONTRACT_DRAFT, location: "朝阳区"}
→ 用户继续: "那个地方租金多少？"
→ Entity Memory 解析: "那个地方" → "朝阳区"
→ Session Memory 恢复上下文继续执行
```

### 6.4 Entity Memory（实体记忆）

从对话中提取的实体信息，支持指代消解：

| 数据字段 | 说明 | 示例 |
|----------|------|------|
| `entity_type` | 实体类型 | location, person, company, amount |
| `entity_value` | 实体值 | "北京朝阳区" |
| `first_seen_at` | 首次提及 | session_45 + msg_12 |
| `mention_count` | 被提及次数 | 3 |

**指代消解示例：**
```
用户: "我要租朝阳区100平米的房子"  → 提取: location=朝阳区
用户: "那个地方租金多少？"         → "那个地方" → 朝阳区
用户: "还要包含押金"              → 继续使用朝阳区上下文
```

### 6.5 User Profile（用户画像）

用户偏好和历史行为：

| 数据字段 | 说明 | 示例 |
|----------|------|------|
| `preferred_contract_types` | 常起草的合同类型 | ["rental", "labor"] |
| `recent_drafts` | 最近起草的合同 | [draft_45, draft_48] |
| `total_drafts` | 累计起草数 | 12 |
| `last_active` | 最后活跃时间 | 2026-04-12 |

### 6.6 Vector Store（向量存储）

### 6.8 记忆读写策略

| 操作 | 触发时机 | 处理 |
|------|----------|------|
| **写入Session** | 每轮对话结束 | 更新Redis + 定期落SQLite |
| **写入Entity** | 识别到新实体时 | 插入或更新mention_count |
| **写入Vector** | 对话结束时 | 整会话向量化存储 |
| **读取Session** | 用户发送消息时 | 优先Redis，miss则查SQLite |
| **检索Vector** | 涉及历史时 | 语义相似度搜索 |

### 6.9 记忆过期策略

| 记忆类型 | 过期条件 | 行为 |
|----------|----------|------|
| Session | 会话结束 > 30天 | 删除 |
| Entity | 30天无提及 | 删除 |
| Vector | 90天无访问 | 归档或删除 |
| User Profile | 用户注销 | 符合隐私合规要求删除 |

## 9. 合同模板格式要求

Agent 要正确调用模板，需要模板满足特定格式要求。

### 9.1 模板格式规范

**推荐格式：Markdown + 占位符**

| 格式 | AI可解析 | 字段填充 | 复杂性 | 推荐度 |
|------|----------|----------|--------|--------|
| Word/PDF | ❌ | ❌ | - | 不推荐 |
| 富文本 | ⚠️ | ⚠️ | 中 | 勉强可做 |
| **Markdown + 占位符** | ✅ | ✅ | 低 | **强烈推荐** |
| 纯JSON | ✅ | ✅ | 中 | 可选方案 |

### 9.2 占位符规范

```
{{field_name}}           # 标准字段
{{field_name?}}          # 可选字段
{{field_name:default}}   # 带默认值
```

### 9.7 AI 解析模板

```python
import re

def extract_placeholders(markdown_content: str) -> list[str]:
    """从Markdown模板中提取占位符"""
    pattern = r'\{\{(\w+)(?::\w+)?\}\}'
    return re.findall(pattern, markdown_content)

def validate_template(markdown_content: str, defined_fields: list[str]) -> dict:
    """验证模板中的占位符是否都有对应字段定义"""
    placeholders = extract_placeholders(markdown_content)
    missing = set(placeholders) - set(defined_fields)

    return {
        "valid": len(missing) == 0,
        "placeholders": placeholders,
        "missing_fields": list(missing)
    }
```
