## 1. 设计概述

### 1.1 设计目标

支持 B端用户完全自定义合同模板字段，无需编写代码，通过界面操作即可定义任意结构的合同。

### 1.2 核心问题与解决方案

| 核心问题 | 解决方案 |
|----------|----------|
| 不同合同字段不同 | 每个模板独立定义字段 |
| 字段太多难以管理 | 分组管理（甲方/乙方/条款/...） |
| 占位符位置不固定 | `template_snippet` 字段记录占位符位置 |
| 相似字段重复定义 | 字段组件库（Preset）去重复用 |
| 字段联动条件 | `conditions` 条件规则 |

## 2. 核心概念

### 2.1 概念层级

```
┌─────────────────────────────────────────────────────────────┐
│                        合同类型                               │
│                   (RENTAL, EMPLOYMENT, ...)                  │
│                           │                                  │
│                           ▼                                  │
│                      合同模板                                 │
│              (房屋租赁合同 v1.0, 劳动合同 v2.0)              │
│                           │                                  │
│                           ▼                                  │
│                      字段分组                                 │
│                (甲方信息, 乙方信息, 条款)                    │
│                           │                                  │
│                           ▼                                  │
│                      具体字段                                │
│         (lessor_name, monthly_rent, lease_start)            │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 模板格式要求

**推荐格式：Markdown + 占位符**

```markdown
# {{contract_title}}

## 甲方信息

**甲方（出租方）**：{{lessor_name}}
**联系电话**：{{lessor_phone}}
**地址**：{{lessor_address}}

## 租赁条款

- **租金**：人民币 {{monthly_rent}} 元/月
- **押金**：人民币 {{deposit}} 元
- **租赁期限**：{{lease_start_date}} 至 {{lease_end_date}}
```

### 2.3 占位符规范

| 格式 | 说明 | 示例 |
|------|------|------|
| `{{field_name}}` | 标准字段 | `{{lessor_name}}` |
| `{{field_name?}}` | 可选字段 | `{{lessor_id_card?}}` |
| `{{field_name:default}}` | 带默认值 | `{{payment_method:monthly}}` |

## 5. 字段组件库

### 5.1 组件库设计

组件库用于字段去重复用，支持 B端用户快速添加字段。

```
┌─────────────────────────────────────────────────────────────┐
│                    字段组件库（Preset）                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  person_name           ← 基础组件（可复用）                   │
│  ├── field_type: text                                       │
│  ├── validation: {min: 2, max: 50}                        │
│  └── used_by: [房东姓名, 租客姓名, 员工姓名, 卖方姓名...]    │
│                                                             │
│  phone_number                                                │
│  ├── field_type: phone                                      │
│  ├── validation: {pattern: "^1[3-9]\\d{9}$"}              │
│  └── used_by: [房东电话, 租客电话, 联系人电话]              │
│                                                             │
│  currency_amount                                             │
│  ├── field_type: currency                                    │
│  ├── validation: {min: 0}                                  │
│  └── used_by: [租金, 押金, 工资, 总价]                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 组件匹配逻辑

```python
def find_matching_preset(field_type: str, validation: dict) -> Preset | None:
    """
    根据字段类型和验证规则匹配预设组件
    返回最匹配的那个（验证规则覆盖最全的）
    """
    # 1. 先按 field_type 筛选
    candidates = [p for p in presets if p.field_type == field_type]

    # 2. 按 validation 相似度排序
    def similarity(p):
        overlap = len(set(validation.keys()) & set(p.validation.keys()))
        return overlap

    candidates.sort(key=similarity, reverse=True)
    return candidates[0] if candidates else None
```

### 5.3 组件Hash去重

```json
{
  "preset_id": "person_name_v1",
  "name": "人名",
  "field_type": "text",
  "validation": {"min": 2, "max": 50},
  "hash": "sha256(person_name+text+min:2+max:50)",
  "usage_count": 156,
  "templates": ["模板A", "模板B", "模板C"]
}
```

### 5.4 组件类型

| field_type | 显示组件 | 验证规则 |
|------------|----------|----------|
| `text` | 单行文本框 | min_length, max_length, pattern |
| `textarea` | 多行文本框 | min_length, max_length |
| `number` | 数字输入框 | min, max |
| `currency` | 金额输入（带单位） | min, max |
| `date` | 日期选择 | min_date, max_date |
| `daterange` | 日期范围 | - |
| `select` | 下拉选择 | options |
| `multiselect` | 多选 | options |
| `radio` | 单选按钮 | options |
| `checkbox` | 复选框 | options |
| `file` | 文件上传 | accept, max_size |
| `phone` | 电话输入 | pattern |
| `idcard` | 身份证 | pattern |
| `email` | 邮箱 | pattern |
| `address` | 地址（省市区联动） | - |

## 6. 字段分组管理

### 6.1 分组结构

```
合同模板
├── 甲方信息（group_order: 1）
│   ├── lessor_name (field_order: 1)
│   ├── lessor_phone (field_order: 2)
│   └── lessor_address (field_order: 3)
├── 乙方信息（group_order: 2）
│   ├── lessee_name (field_order: 1)
│   ├── lessee_phone (field_order: 2)
│   └── lessee_address (field_order: 3)
└── 租赁条款（group_order: 3）
    ├── property_address (field_order: 1)
    ├── monthly_rent (field_order: 2)
    ├── deposit (field_order: 3)
    ├── lease_start_date (field_order: 4)
    ├── lease_end_date (field_order: 5)
    └── payment_method (field_order: 6)
```

### 6.2 分组定义

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | int | 分组ID |
| `template_id` | int | 关联的模板 |
| `group_name` | string | 分组名称（甲方信息/乙方信息/条款） |
| `group_order` | int | 分组排序 |
| `fields` | list | 字段列表 |

### 6.3 分组操作

| 操作 | 说明 |
|------|------|
| 创建分组 | 输入分组名称 |

## 7. 条件字段

### 7.1 条件规则定义

```json
{
  "field_name": "deposit",
  "display_name": "押金",
  "conditions": [
    {
      "field": "contract_type",
      "operator": "equals",
      "value": "RENTAL",
      "action": "show"
    },
    {
      "field": "contract_type",
      "operator": "in",
      "value": ["RENTAL", "LEASE"],
      "action": "required"
    }
  ]
}
```

### 7.2 操作符

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `equals` | 等于 | contract_type equals "RENTAL" |
| `not_equals` | 不等于 | status not_equals "draft" |
| `in` | 包含 | contract_type in ["RENTAL", "LEASE"] |
| `not_in` | 不包含 | contract_type not_in ["GIFT"] |
| `contains` | 包含字符串 | name contains "租赁" |
| `greater_than` | 大于 | amount > 1000 |
| `less_than` | 小于 | amount < 10000 |

### 7.3 动作

| 动作 | 说明 |
|------|------|
| `show` | 显示字段 |
| `hide` | 隐藏字段 |
| `required` | 设为必填 |
| `optional` | 设为可选 |
| `readonly` | 设为只读 |
| `default` | 设置默认值 |

### 7.4 条件组合

支持多条件组合（AND/OR）：

```json
{
  "field_name": "trial_period",
  "conditions": [
    {
      "logic": "AND",
      "rules": [
        {"field": "contract_type", "operator": "equals", "value": "EMPLOYMENT"},
        {"field": "has_trial", "operator": "equals", "value": true}
      ],
      "action": "show"
    }
  ]
}
```

## 8. 数据库设计

### 8.1 表结构

```sql
-- 合同类型表
CREATE TABLE contract_types (
    id SERIAL PRIMARY KEY,
    code VARCHAR(50) UNIQUE NOT NULL,      -- RENTAL, EMPLOYMENT, etc.
    name VARCHAR(100) NOT NULL,            -- 租赁合同, 劳动合同
    description TEXT,
    sort_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 合同模板表
CREATE TABLE contract_templates (
    id SERIAL PRIMARY KEY,
    type_id INT REFERENCES contract_types(id),
    name VARCHAR(200) NOT NULL,
    description TEXT,
    content TEXT,                           -- Markdown + 占位符
    version VARCHAR(20) DEFAULT '1.0',
    status VARCHAR(20) DEFAULT 'draft',     -- draft, published, archived
    field_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 字段分组表
CREATE TABLE field_groups (
    id SERIAL PRIMARY KEY,
    template_id INT REFERENCES contract_templates(id) ON DELETE CASCADE,
    group_name VARCHAR(100) NOT NULL,
    group_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 模板字段表
CREATE TABLE template_fields (
    id SERIAL PRIMARY KEY,
    template_id INT REFERENCES contract_templates(id) ON DELETE CASCADE,
    group_id INT REFERENCES field_groups(id) ON DELETE SET NULL,

    field_name VARCHAR(100) NOT NULL,       -- lessor_name（唯一标识）
    display_name VARCHAR(100) NOT NULL,      -- 房东姓名（显示用）
    field_type VARCHAR(50) NOT NULL,         -- text, number, select...

    field_order INT DEFAULT 0,              -- 字段在组内排序

    required BOOLEAN DEFAULT FALSE,
    read_only BOOLEAN DEFAULT FALSE,
    hidden BOOLEAN DEFAULT FALSE,

    placeholder VARCHAR(200),
    description TEXT,
    default_value TEXT,

    validation_rules JSON,                   -- {"min": 0, "max": 100, "pattern": "..."}
    options JSON,                           -- select/radio的选项

    conditions JSON,                        -- 条件显示/必填规则

    preset_id INT REFERENCES field_presets(id),  -- 引用的组件

    template_snippet TEXT,                  -- 正文片段：甲方：{{lessor_name}}

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    UNIQUE(template_id, field_name)
);

-- 字段组件预设表
CREATE TABLE field_presets (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,             -- 组件名称
    field_type VARCHAR(50) NOT NULL,         -- text, number, select...
    validation_rules JSON,                   -- 验证规则
    hash VARCHAR(64) UNIQUE NOT NULL,       -- 去重hash
    usage_count INT DEFAULT 0,              -- 使用次数
    is_system BOOLEAN DEFAULT FALSE,        -- 是否系统内置
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 字段选项预设表
CREATE TABLE field_options (
    id SERIAL PRIMARY KEY,
    field_id INT REFERENCES template_fields(id) ON DELETE CASCADE,
    option_value VARCHAR(100) NOT NULL,
    option_label VARCHAR(100) NOT NULL,
    sort_order INT DEFAULT 0
);
```

### 8.2 索引设计

```sql
-- 模板字段按模板+分组索引
CREATE INDEX idx_template_fields_template_group ON template_fields(template_id, group_id);

-- 组件按hash索引
CREATE INDEX idx_field_presets_hash ON field_presets(hash);

-- 模板按类型索引
CREATE INDEX idx_contract_templates_type ON contract_templates(type_id);
```

## 12. 术语表

| 术语 | 说明 |
|------|------|
| **Contract Type** | 合同类型，如 RENTAL、EMPLOYMENT |
| **Contract Template** | 合同模板，包含字段定义和正文 |
| **Field Group** | 字段分组，如甲方信息、乙方信息 |
| **Template Field** | 模板字段，对应正文中的占位符 |
| **Field Preset** | 字段组件预设，用于去重复用 |
| **Field Type** | 字段类型，如 text、number、select |
| **Validation Rules** | 验证规则，如必填、长度、格式 |
| **Conditions** | 条件规则，字段的显示/必填条件 |
| **Template Snippet** | 正文片段，字段在正文中的位置 |
| **Placeholder** | 占位符，模板正文中用 `{{field_name}}` 标记 |
