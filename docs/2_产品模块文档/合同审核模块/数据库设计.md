# 合同审核模块 - 数据库设计

## 1. contract_review_reports - AI审核报告表

```sql
CREATE TABLE contract_review_reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    contract_id INTEGER NOT NULL,
    risk_summary JSON,                       -- {"high": 2, "medium": 5, "low": 3}
    risk_items JSON,                          -- 风险点详情列表
    ai_model VARCHAR(50),                     -- 使用的模型
    rule_doc_id INTEGER,                      -- 使用的规则文档ID（可选）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (contract_id) REFERENCES contracts(id)
);
```

### risk_summary 示例

```json
{
  "high": 2,
  "medium": 5,
  "low": 3
}
```

### risk_items 示例


## 2. contract_review_items - 审核确认记录表

```sql
CREATE TABLE contract_review_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    report_id INTEGER NOT NULL,
    risk_item_id INTEGER NOT NULL,           -- 对应报告中的风险点ID
    confirmed_by INTEGER,                     -- 确认人ID
    confirmed_at DATETIME,                     -- 确认时间
    reviewer_comment VARCHAR(500),            -- 审核意见
    status VARCHAR(20) DEFAULT 'pending',     -- pending/confirmed/modified
    FOREIGN KEY (report_id) REFERENCES contract_review_reports(id)
);
```

## 3. contract_review_logs - 审核日志表

```sql
CREATE TABLE contract_review_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    contract_id INTEGER NOT NULL,
    action VARCHAR(50) NOT NULL,              -- ai_review/assign/start/pass/reject
    operator_id INTEGER NOT NULL,
    operator_role VARCHAR(20),                -- admin/staff
    detail JSON,                             -- 操作详情
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (contract_id) REFERENCES contracts(id)
);
```

## 4. 风险等级定义

| 等级 | 颜色 | 说明 |
|------|------|------|
| high | 红色 | 严重风险，必须修改 |
| medium | 橙色 | 中等风险，建议修改 |
| low | 黄色 | 轻微风险，可选修改 |

## 5. 风险项类型定义

| type | 说明 | 来源 |
|------|------|------|
| invalid_field | 无效字段值 | 预检测 |
| missing_field | 缺失字段（空白占位符） | 预检测 |
| risk_clause | 风险条款 | AI检测 |
| missing_clause | 缺失条款 | AI检测 |
| date_error | 日期错误 | AI检测 |
| comprehensive | 综合评估 | AI检测 |

## 7. 两层检测机制

### 第一层：代码预检测（确定性规则）

**无效字段值检测**：

| 检测类型 | 规则说明 | 示例 | 优先级 |
|----------|----------|------|--------|
| 纯数字序列 | 非金额字段填写纯数字 | lessee_name: "123123213" | medium |
| 电话格式错误 | 必须11位，以1开头 | contact_phone: "131231" (6位) | high |
| 身份证格式错误 | 必须是15或18位 | lessor_id: "1212121" (7位) | high |
| 日期格式错误 | 符合YYYY-MM-DD等标准格式 | lease_start: "123123" | high |
| 英文重复字符 | 连续重复的英文字母 | xxx, aaa, test | medium |
| 测试占位符 | 常见测试词汇 | 张三, 李四, 测试, 随便 | medium |
| 临时占位符 | 表示未填写的占位词 | 待填写, 暂无, 待定, TBD | medium |

**缺失字段检测**：

| 检测模式 | 正则表达式 | 匹配示例 |
|----------|------------|----------|
| 字段名+空白 | `字段名：__________` | 男方：__________ |
| 日期格式空白 | `_____年___月___日` | 出生日期、签订日期 |

### 第二层：AI智能审核（模糊判断）

- 风险条款识别：不公平条款、违法条款、模糊条款
- 缺失条款检测：争议解决、保密条款等
- 综合评估：结构合理性、表述清晰度
