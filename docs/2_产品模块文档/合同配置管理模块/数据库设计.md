# 合同配置管理模块 - 数据库设计

## 1. contract_categories - 合同类型表

```sql
CREATE TABLE contract_categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(50) UNIQUE NOT NULL,        -- real_estate_lease, loan, labor
    name VARCHAR(100) NOT NULL,               -- 房产租赁、借贷、劳动用工
    description VARCHAR(500),
    sort_order INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 预设合同类型

| 类型编码 | 类型名称 | 说明 |
|----------|----------|------|
| real_estate_lease | 房产租赁 | 房屋、商铺等租赁合同 |
| loan | 借贷 | 个人或企业间借款合同 |
| labor | 劳动用工 | 劳动合同、劳务协议 |
| trade | 买卖交易 | 商品、房产等买卖合同 |
| marriage | 婚姻家庭 | 婚前协议、离婚协议等 |

## 2. field_library - 字段库表

```sql
CREATE TABLE field_library (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(50) UNIQUE NOT NULL,        -- deposit, rent_price
    name VARCHAR(100) NOT NULL,             -- 押金、月租金
    field_type VARCHAR(20) DEFAULT 'text',   -- text/date/number/select
    category_id INTEGER,                     -- NULL=通用字段，否则为专属字段
    validation_rule VARCHAR(200),           -- 验证规则
    default_value VARCHAR(200),             -- 默认值
    description VARCHAR(500),                -- 帮助提示
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES contract_categories(id)
);
```

### 字段分类规则
- **通用字段**：`category_id = NULL`，所有合同类型可见
- **专属字段**：`category_id` 非NULL，仅对应合同类型可见

### 预设通用字段

| 字段编码 | 字段名称 | 字段类型 |
|----------|----------|----------|
| party_a | 甲方 | text |
| party_b | 乙方 | text |
| amount | 金额 | number |
| date | 日期 | date |
| start_date | 开始日期 | date |
| end_date | 结束日期 | date |
| phone | 联系电话 | text |
| address | 地址 | text |
| id_number | 身份证号 | text |

## 3. templates - 模板表

```sql
CREATE TABLE templates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,              -- 模板名称
    file_path VARCHAR(255) NOT NULL,         -- 文件路径
    file_type VARCHAR(20) NOT NULL,          -- pdf/word/md/text
    contract_type VARCHAR(50) NOT NULL,       -- 合同类型
    content TEXT,                             -- 解析后的模板内容
    category_id INTEGER,                      -- 关联合同类型
    content_html TEXT,                       -- 转换后的HTML（保留格式）
    content_text TEXT,                       -- 纯文本（用于检索）
    status VARCHAR(20) DEFAULT 'draft',       -- draft/active/archived
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES contract_categories(id)
);

-- 索引
CREATE INDEX idx_templates_type ON templates(contract_type);
CREATE INDEX idx_templates_status ON templates(status);
CREATE INDEX idx_templates_category ON templates(category_id);
```

## 4. template_field_mappings - 模板字段映射表

```sql
CREATE TABLE template_field_mappings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    template_id INTEGER NOT NULL,
    source_type VARCHAR(20) NOT NULL,       -- variable(【】) / blank(下划线)
    source_text VARCHAR(200),                -- 【甲方名称】或留白上下文
    position_index INTEGER,                  -- 位置索引（用于定位）
    field_id INTEGER,                        -- 映射的字段ID，NULL表示不填写
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (template_id) REFERENCES templates(id),
    FOREIGN KEY (field_id) REFERENCES field_library(id)
);
```

### source_type 说明

| 值 | 说明 | 配置要求 |
|----|------|----------|
| variable | 变量标记【变量名】 | **必须**映射到字段 |
| blank | 下划线留白 | 可选映射或不填写 |

## 5. 模板格式规范

### 文件格式要求
- 文件格式：Word文档（.docx）
- 变量标记：使用【变量名】格式，变量名建议清晰明确
- 留白位置：使用连续下划线标记，建议至少5个下划线（_____）
- 禁止在【】括号内使用下划线

### 示例

```
甲方名称：【甲方名称】
乙方名称：【乙方名称】
月租金：【月租金】

甲方：_______________
乙方：_______________
```
