## 2. 核心流程设计

### 2.1 对话流程

```
用户发送消息 → API网关 → 意图识别 → 法律范围判断 → 意图分流 → 响应生成 → SSE推送
```

### 2.2 意图识别流程

```
用户消息
    │
    ├──→ 关键词匹配 ──→ 命中 ──→ 返回意图
    │
    ├──→ 正则匹配 ──→ 命中 ──→ 返回意图
    │
    ├──→ 语义相似度 ──→ 阈值以上 ──→ 返回意图
    │
    └──→ LLM辅助判断 ──→ 返回意图 + 置信度
```

### 2.3 合同生成流程

```
用户描述需求 → 创建合同会话 → 字段收集(逐个提问) → 模板填充 → AI润色 → 用户确认 → 导出PDF
```

### 2.4 合同审核流程

```
提交审核 → AI预检测 → AI智能审核 → 生成报告 → 人工确认 → 主管审批 → 完成
```

## 5. 安全设计

### 5.1 认证授权
- **JWT Token**：有效期7天
- **角色区分**：C端用户 / B端admin/主管/律师
- **权限控制**：API层面校验角色权限

### 5.2 数据安全
- **密码存储**：bcrypt加密
- **敏感信息**：身份证、银行卡等自动脱敏
- **审计日志**：关键操作全部记录

### 5.3 敏感信息掩码规则

| 类型 | 规则 | 示例 |
|------|------|------|
| 身份证号 | 前3后4可见 | 110***********1234 |
| 银行卡号 | 前4后4可见 | 6222****1234 |
| 地址 | 保留省市区 | 北京市朝阳区*** |

## 6. 部署架构

### 6.1 Docker Compose

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
    environment:
      - SECRET_KEY=${SECRET_KEY}
```

### 6.2 数据目录

```
/data/
├── templates/          # 合同模板
├── documents/         # 案例、审核规则等
├── rag_index/         # Chroma向量索引
├── contracts/         # 生成的合同文档
└── logs/              # 审计日志
```

## 7. 扩展性设计

### 7.1 后续版本规划

| 版本 | 功能扩展 | 架构调整 |
|------|----------|----------|
| V1.1 | 合同审核工作流 | 新增审核模块 |
| V1.x | B端智能体对话 | 复用Agent模块 |
| V2.0 | 多租户 | 数据库迁移PostgreSQL |

### 7.2 模块解耦
- **配置服务**：独立配置模块，支持热更新
- **Agent模块**：插件化设计，支持新增意图处理器
- **文档处理**：独立解析模块，支持新增文档格式
