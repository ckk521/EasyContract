## 一、产品概述

### 1.1 产品定位
面向企业法务、中小企业、律所的智能合同审核平台，通过 AI + 人工协同，提供合同审核、风险识别、修改建议、合同生成等一站式服务。

### 1.2 核心价值
| 用户群 | 核心价值 |
|--------|----------|
| 企业法务 | 效率提升、风险控制、标准化流程 |
| 中小企业 | 降低法务成本、专业审核能力 |
| 律所 | 提升产能、智能辅助决策 |
| C端用户 | 便捷法律咨询、透明进度查询 |

### 1.3 MVP目标
- 完成核心审核流程闭环
- 验证 AI 审核准确率
- 打通 B端管理后台 + C端智能体

### 3.2 模块详细说明

#### 3.2.1 B端管理后台

##### A. 系统管理

| 功能 | 描述 |
|------|------|
| 用户管理 | 新增/编辑/禁用用户，分配角色 |
| 角色权限 | 角色定义、权限配置（预置，MVP暂不支持自定义） |
| 配额管理 | 为企业/用户分配审核次数，查看使用记录，欠费报警 |

##### B. 合同管理

| 功能 | 描述 |
|------|------|
| 合同列表 | 所有合同状态总览：待审核、审核中、待审批、已完成、已驳回 |
| 审核任务 | 法务专员的待办任务，支持领取/分配 |
| 审核报告 | AI审核报告 + 人工修正，风险清单，修改建议 |
| 模板管理 | 合同模板库（MVP预置2-3个模板） |

##### C. 风险规则库

| 功能 | 描述 |
|------|------|
| 规则列表 | 查看所有风险规则，按类别/等级筛选 |
| 规则编辑 | 新增/编辑风险规则（结构化表单） |
| RAG知识库 | 上传法律文档，向量化存储，支持检索增强 |
| 实时更新 | **提供入口随时更新规则，支持增量添加和修改** |

##### D. 系统异常校验

| 校验场景 | 提示信息 |
|----------|----------|
| 未配置合同模板 | "请先配置合同模板后再生成合同" |
| 用户配额为0且欠费 | "您的配额已用完，请联系管理员充值" |
| 风险规则库为空 | "请先添加风险规则后再进行审核" |
| 法务人员未配置 | "请先配置法务人员后再分配任务" |

##### D. 法务人员管理

| 功能 | 描述 |
|------|------|
| 人员列表 | 法务专员、Manager列表 |
| 画像标签 | 设置专长标签：合同类型、行业领域、风险等级偏好 |
| 工作量统计 | 当前任务数、历史处理量、平均处理时长 |

## 五、风险规则体系

### 5.1 风险规则结构

```json
{
  "rule_id": "R001",
  "rule_name": "违约金比例过高",
  "category": "违约责任",
  "risk_level": "high",
  "description": "违约金超过合同总金额30%可能被法院调低",
  "legal_basis": "民法典第585条",
  "detection_pattern": "违约金.*超过|违约金比例.*%",
  "threshold": {
    "type": "percentage",
    "value": 30,
    "compare": ">"
  },
  "suggestion": "建议调整为不超过合同总金额的20%",
  "examples": [
    "违约金为合同总金额的50%",
    "违约方需支付合同价款40%的违约金"
  ]
}
```

### 5.2 风险等级定义

| 等级 | 标识 | 颜色标记 | 含义 | 处理要求 |
|------|------|----------|------|----------|
| 高 | HIGH | 🔴 红色 | 重大法律风险，可能导致无效/纠纷 | 必须修改 |
| 中 | MEDIUM | 🟡 黄色 | 存在风险，建议优化 | 建议修改 |
| 低 | LOW | 🟢 绿色 | 轻微问题，仅供参考 | 提示参考 |

> **风险等级通过RAG技术匹配**，系统根据合同内容与风险规则库进行语义检索，自动判定风险等级。

### 5.3 规则分类（MVP阶段）

| 分类 | 规则数量（预估） | 示例 |
|------|------------------|------|
| 合同主体 | 20+ | 主体资格、授权限制 |
| 标的条款 | 30+ | 标的不明、权属争议 |
| 价款支付 | 40+ | 付款方式、发票要求 |
| 履行期限 | 25+ | 期限不明、延期责任 |
| 违约责任 | 50+ | 违约金、赔偿限额 |
| 争议解决 | 30+ | 管辖法院、仲裁条款 |
| 保密条款 | 20+ | 保密范围、泄密责任 |
| 知识产权 | 25+ | 权属约定、侵权风险 |
| 其他条款 | 60+ | 不可抗力、解除条件 |
| **合计** | **~300条** | |

### 5.4 修改建议生成机制

修改建议由大模型生成，需配置专用的提示词模板和Few-Shot示例。

#### 提示词模板结构

```
你是一位专业的法律顾问，擅长合同审核和风险规避。

## 任务
根据以下风险项，生成具体的修改建议。

## 风险项
- 类型: {risk_category}
- 等级: {risk_level}
- 问题描述: {description}
- 原条款内容: {original_clause}
- 法律依据: {legal_basis}

## 输出要求
1. 指出具体问题所在
2. 说明法律风险
3. 提供修改后的条款建议
4. 保持条款的完整性和专业性

## 输出格式
**问题分析**: [问题说明]
**法律风险**: [风险解释]
**修改建议**: [修改后的条款]
```

#### Few-Shot 示例

**示例1: 违约金过高**

```
输入:
- 原条款: "任何一方违约，需向守约方支付合同总金额50%的违约金"
- 风险: 违约金比例过高，可能被法院调整

输出:
**问题分析**: 违约金比例为合同总金额的50%，远超司法实践中通常支持的合理范围（通常不超过30%）。
**法律风险**: 根据《民法典》第585条，约定的违约金过分高于造成的损失的，人民法院或者仲裁机构可以根据当事人的请求予以适当减少。
**修改建议**: "任何一方违约，需向守约方支付合同总金额20%的违约金。如违约金不足以弥补实际损失的，可请求增加；如违约金过分高于实际损失的，可请求适当减少。"
```

**示例2: 争议解决条款不明**

```
输入:
- 原条款: "双方发生争议协商不成的，通过法律途径解决"
- 风险: 争议解决方式不明确，可能导致管辖权争议

输出:
**问题分析**: 条款仅表述"通过法律途径解决"，未明确是诉讼还是仲裁，也未约定管辖法院。
**法律风险**: 可能导致双方就管辖权产生争议，增加诉讼成本和时间成本。
**修改建议**: "双方因本合同发生争议协商不成的，任何一方均可向原告方所在地人民法院提起诉讼。" 或 "双方因本合同发生争议协商不成的，提交XX仲裁委员会仲裁。"
```

### 9.4 修改建议应用方式

```
修改建议应用流程：

用户查看建议 → 通过智能体反馈 → Agent分析可行性
                                    ↓
                        ┌───────────┴───────────┐
                        │                       │
                   Agent同意修改          Agent认为不宜修改
                        │                       │
                        ▼                       ▼
                   自动修改条款           提示风险原因
                        │                       │
                        ▼                       ▼
                   标记修改记录      用户坚持修改 → 备注风险后修改
```

**核心原则**：
- 用户不能直接修改条款
- 所有修改通过Agent审核
- Agent不同意但用户坚持 → 强制备注风险

```
┌─────────────────────────────────────────────────────────────┐
│                      合同状态流转                            │
└─────────────────────────────────────────────────────────────┘

                    ┌─────────────┐
                    │   待初审     │
                    └──────┬──────┘
                           │ AI开始审核
                           ▼
                    ┌─────────────┐
                    │  AI审核中    │

                    ┌─────────────┐

                           │ 提交审批            │ Manager驳回
                           ▼                     │
                    ┌─────────────┐              │
                    │   待审批     │──────────────┘
                    └──────┬──────┘
                           │ 审批通过
                           ▼
                    ┌─────────────┐
                    │   已通过     │
                    └──────┬──────┘

                    └──────┬──────┘       │
                           │              │ 用户提出修改
              ┌────────────┼──────────────┘
              │            │
              ▼            ▼

        └──────────┘  └──────────────┘
```

#### 状态定义

| 状态 | 说明 | 可流转至 |
|------|------|----------|
| 待初审 | 合同刚上传，等待AI审核 | AI审核中 |
| AI审核中 | AI正在进行审核 | 待分配 |
| 待分配 | AI审核完成，等待分配法务 | 审核中 |
| 审核中 | 法务专员处理中 | 待审批 |
| 待审批 | 等待Manager审批 | 已通过、审核中（驳回） |
| 已通过 | Manager审批通过 | 用户确认中 |
| 用户确认中 | 等待用户确认 | 已完成、重新审核 |
| 已完成 | 用户已确认 | - |
